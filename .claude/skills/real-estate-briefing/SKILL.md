# Real Estate Briefing

강남/서초/송파 추적 단지의 호가 vs 실거래가 괴리율 브리핑 워크플로우.

## 추적 대상

13개 단지 (3개 구), `re_complexes.is_tracked = true`

| 구 | 단지 |
|---|------|
| 강남구 | 래미안대치팰리스, 은마, 도곡렉슬, 개포래미안포레스트 |
| 서초구 | 래미안원베일리, 반포자이, 아크로리버파크, 래미안퍼스티지 |
| 송파구 | 잠실엘스, 리센츠, 트리지움, 헬리오시티, 파크리오 |

## 기본 브리핑 범위

- **평형대**: CEO 요청에 따라 (기본: 30평대만. 요청 시 20/40/50평대 추가)
- **구**: CEO 요청에 따라 (기본: 전체 3개 구. 특정 구 지정 가능)
- **매매/전세**: 기본 매매. CEO 요청 시 전세 추가

## Step 1: 데이터 조회

### MCP 도구 사용 순서

1. **`re_get_listing_gap`** — 핵심 도구. 호가 vs 실거래가 괴리율
   - `trade_type`: "매매" (기본) / "전세"
   - 반환값: 단지별, 평형대별 실거래 평당가, 최저호가 평당가, 괴리율, 매물수

2. **`re_list_listings`** — 실제 호가 레인지 확인 (보조)
   - `complex_name`: 단지명
   - `trade_type`: "매매"
   - `limit`: 10~20
   - 반환값: 개별 매물의 호가, 면적, 층, 방향

3. **`re_get_summary`** — 전체 시장 요약 (선택)
   - `district`: 구 필터
   - 반환값: 추적단지수, 평균 평당가, 평균 괴리율

## Step 2: 단위 규칙 (필수!)

### 평당가 표기
- **억/평 단위 사용**: `1.22억/평`, `0.92억/평`
- DB 원본은 만원 단위 (예: 12,249만) → `1.22억`으로 변환
- ❌ 절대 금지: `12,249만/평` — CEO에게 직관적이지 않음

### 총 호가 표기
- **억원 단위 사용**: `36.9억`, `45.2억`
- 계산: 평당가(억) × 평수 = 총 호가(억)
- 예: 1.08억/평 × 34평 = 36.7억
- ❌ 절대 금지: 10으로 나누는 실수 (4.17억 → 틀림, 41.7억이 맞음)

### 변환 공식
```
DB 평당가(만원) → 억 표기: 값 / 10,000 → "X.XX억/평"
총 호가(억): 평당가(억) × 평수
평수: 공급면적(㎡) / 3.3058
```

## Step 3: 브리핑 포맷

### 구별로 섹션 분리 (CEO 요청한 구만)

```
## 🏠 [구이름] [평형대] — [한 줄 요약]

| 단지 | 실거래 평당가 | 최저호가 평당가 | 괴리율 | 매물수 |
|------|------------|-------------|-------|-------|
| 래미안대치팰리스 | **1.22억**/평 | **1.08억**/평 | **-11.5%** | 301건 |
| ... | ... | ... | ... | ... |

**추정 총 호가 (최저~최고):**
- **래미안대치팰리스** 34평 기준: 약 36.9억 ~ 52.5억
- ...

**[평형대] 요약**: [매수자/매도자 우위 판단 + 핵심 포인트]
```

### 한 줄 요약 패턴
- 전 단지 호가 < 실거래가 → "매수자 우위"
- 전 단지 호가 > 실거래가 → "매도자 우위"
- 혼재 → "단지별 차별화"

### 괴리율 해석
| 괴리율 | 의미 |
|--------|------|
| -10% 이상 | 매도 압력 매우 강함 |
| -5% ~ -10% | 매도 압력 있음 |
| -3% ~ -5% | 약한 약세 |
| -3% ~ +3% | 균형 |
| +3% ~ +5% | 약한 강세 |
| +5% ~ +10% | 매수 경쟁 |
| +10% 이상 | 과열 |

## Step 4: 종합 판단

브리핑 마지막에 종합 판단 추가:
- 전체 시장 방향성 (약세/강세/혼조)
- 평형대별 차이 (대형 vs 소형)
- 매물 물량 수준 (풍부/적정/부족)
- 매수/매도 시사점

## 멀티 메시지 분할

텔레그램 브리핑 시 `---SPLIT---` 사용:
- 구별로 분할 (강남 → SPLIT → 서초 → SPLIT → 송파)
- 평형대별로 분할 (30평대 → SPLIT → 40평대)
- 종합 판단은 마지막 SPLIT 후

## 데이터 없는 경우

특정 평형대/단지에 데이터가 없으면:
- "유의미한 거래 데이터 없음" 표기
- 해당 행 생략 (빈 칸으로 채우지 않음)
- 40평대 이상은 데이터 부족 가능성 높음 — 미리 안내
